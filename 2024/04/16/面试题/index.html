# js 中原生回调函数、promise、async/await 的用法和区别

在 JavaScript 中，原生回调函数、Promise、以及 async/await 是处理异步操作的三种主要方法，它们各自解决了不同的问题，并且在易用性、可读性和错误处理等方面有所不同。

  1. 原生回调函数：

    • 原生回调函数是 JavaScript 中处理异步操作的最基本方式之一，它通常通过函数参数的方式来实现。
    • 回调函数在异步操作完成后被调用，传递结果或错误。
    • 问题：回调地狱（Callback Hell）是其主要问题之一，当多个异步操作依赖于前一个异步操作的结果时，嵌套的回调函数会使代码变得难以理解和维护。

  2. Promise：

    • Promise 是 ES6 引入的一种解决回调地狱问题的方法，它提供了更加结构化的方式来处理异步操作。
    • Promise 表示一个异步操作的最终完成或失败，并且其结果值是可用的。
    • 通过链式调用 .then() 和 .catch() 方法，可以更清晰地处理异步操作的结果和错误。
    • 问题：仍然需要使用 .then() 和 .catch() 来处理异步操作，有时候可能导致一些嵌套。

  3. async/await：

    • async/await 是 ES8 引入的一种基于 Promise 的异步编程解决方案，它提供了更加简洁和直观的语法来处理异步操作。
    • async 关键字用于定义一个函数是异步的，这样函数内部就可以使用 await 关键字来等待 Promise 的解决或拒绝。
    • await 关键字可以暂停异步函数的执行，直到 Promise 解决，并且可以将解决值作为返回值。
    • async/await 语法使得异步代码看起来更像同步代码，更易读和维护。
    • 可以使用 try/catch 块来捕获异步操作中的错误。
    • 问题：某些情况下，使用 async/await 可能会造成代码执行顺序不明确，需要小心处理。

总的来说，这些方法解决了 JavaScript 中异步操作的问题，并且随着 JavaScript 的发展，async/await 成为了处理异步操作的首选方法，因为它提供了更好的可读性和错误处理机制。
